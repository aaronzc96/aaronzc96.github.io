面试题 05.03. 翻转数位

给定一个32位整数 num，你可以将一个数位从0变为1。请编写一个程序，找出你能够获得的最长的一串1的长度。

示例 1：

输入: num = 1775(11011101111)二进制
输出: 8
示例 2：

输入: num = 7(0111)二进制
输出: 4

来源：力扣（LeetCode）
链接：https://leetcode-cn.com/problems/reverse-bits-lcci

简单题，也是平生最讨厌系列，很难去考虑很全面，加上可爱的贡献者大开脑洞贡献的用例，我只能%%#&

先上代码，再解释思路：

思路其实很简单，就是采用两个指针l和r，在这指针范围内的零的个数不超过2，什么意思？就是l和人之间最多存在1个0，这其实就是紧扣题意给的思路。

  • 首先我们用bitset将num转换为二进制，这里转换后的二进制实际上是从最低位往最高位进行存储的，比如如果是8（1000），存到bitset中实际b[0] = 0, b[1] = 0, b[2] = 0, b[3] = 1

  • 接下来定义几个标志：

    zn存储当前指针内0的个数

    l和r为左右元素的下标（所谓的左右指针）

    zi存储指针范围内的第一个0的索引（主要用来更新左指针）

    ans存储最大长度

  • 接下来就可以遍历整个二进制字符串了

    首先如果当前位置是0，则zn（记录0的个数）要增加，而且如果zi（记录当前指针范围内的第一个0的位置）还是初始的-1即之前从未有0出现，则更新为当前r值

    然后判断zn是否超过1（当前指针范围内0的个数是否大于1），容易知道，如果超过了一定是由于在本次循环中，上一步更新了zn，因此可以断定当前的b[r] == 0

    如果上面的条件满足，那么我们就需要做一下更新的动作：

    1. 首先需要更新一下最大值

    2. 其次需要更新左指针

    3. 将zn修改为1，即更新左指针之后左右指针范围内0的个数

    4. zi由于已经协助更新了l的值，因此此处需要将当前的r赋予zi，记录更新左指针之后，左右指针范围内第一个0的位置

  • 最后不能直接返回ans，意会一下就明白了
